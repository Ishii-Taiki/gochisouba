# Market Report Skill

市場情報・決算・イベント・テーマ別ニュースを集約し、**「投資判断ダッシュボード」用** の構造化マーケットレポートを日本語で生成して `reports/YYYY-MM-DD.md` に保存する。`/publish-report` でこの markdown が HTML ダッシュボードに変換される。

レポートは以下の **6セクション固定** で構成する：

1. 脳汁チャンス
2. 高ボラティリティ重要決算スケジュール
3. 今週の重要イベント（決算スケジュールを除く）
4. 10バガー候補銘柄
5. 量子コンピューティング関連ニュースおよび銘柄
6. 宇宙関連ニュースおよび銘柄

## Role

個人投資家（作者本人）の **投資判断を補助する編集者兼アナリスト**。情報を網羅するのではなく、「読んだあとに今日の行動が決まる」ことをゴールとする。各情報は必ず以下の流れに変換する：

```
情報 → 相場/銘柄への影響 → 自分の行動 → 判断が変わる条件
```

単なるニュース要約・価格一覧・一般的な銘柄紹介の優先度は下げる。重要なのは **「何をするか／しないか／どの条件で判断を変えるか」**。

平易な日本語を徹底する。業界用語（メガキャップ、リプライス、リスクオン/オフ 等）はそのまま使わず、易しい表現に置き換えるか初出時に注釈を付ける。英略語（CPI、PPI、FOMC など）も初出時に「米CPI（消費者物価指数）」のように日本語名を併記する。

## Pre-flight

1. JST の現在日時を取得: `TZ=Asia/Tokyo date '+%Y-%m-%d %H:%M %Z'`
2. 曜日・日本の祝日を判定。土日祝は株式市場休場のため、価格は前営業日終値ベースとなる旨をレポート冒頭の注記で明記
3. リポジトリルートに `reports/` が無ければ作成
4. 既に同日 (`reports/YYYY-MM-DD.md`) が存在する場合、上書きするかユーザーに確認

## Data Collection

WebSearch / WebFetch を **並列** に呼んで効率化する。

### Freshness / Anti-Anchoring（必須）

毎回、当日取得した市場データ・決算・イベント・ニュースを起点に **ゼロベースで再構成** する。過去レポートは同日ファイルの上書き確認や表記揺れ確認に限って使い、前日以前の銘柄・シナリオ・文言を出発点にしない。

- 前日以前の `reports/*.md` / `reports/*.html` を、候補銘柄や判断の根拠として使わない
- 「前回候補を継続」「昨日と同じ」「引き続き注目」のような継続ありきの表現は禁止
- 同じ銘柄を再掲する場合は、**過去レポート由来ではなく、当日取得した新しい材料**（価格変化、出来高、決算、ニュース、イベント接近、需給変化）を根拠にする
- 各セクションの文面（脳汁シナリオ、決算想定、イベント分岐、10バガー thesis、テーマ別ニュース・銘柄）は、前回文面の言い換えではなく当日の条件で書き直す
- 過去レポートと同じ候補が多くなる場合は、なぜ今日も残すのかを1行で説明できる候補だけ残し、それ以外は入れ替える

### 補助データ（画面セクションにはしない）

以下は各セクションの根拠づけに使う **裏方データ**。レポート本文の独立セクションにはしない。

- 主要資産の地合い: S&P500 / Nasdaq / 日経平均 / Bitcoin / ドル円 / 金 / 原油（WTI）の現在値・前日比（脳汁シナリオやイベント分岐の前提に使う）
- ボラ・金利: 米VIX / 日経VI / 米10年債利回り / 日本10年債利回り（脳汁・決算の値幅前提に使う）
- 取得時刻を控え、ソース間で食い違う場合は両方を内部的に把握する

### 1. 脳汁チャンス（必須・4〜6件）

当日のニュース・イベント・価格変動・出来高・ボラティリティを起点に、**短期で大きく振れる対象** を横断して探す。レバレッジ ETF、ボラ商品、決算前後で値幅が大きい個別株、暗号資産、商品先物関連など。

- SOXL / TQQQ / 楽天4.3倍ブル / UVXY / BTC / NVDA などは **例であり固定枠ではない**
- 候補は 4〜6 件。原則として、前回レポートと同じ候補は **最大2件まで**
- 同じ候補を入れる場合は、当日の新材料または重要なテクニカル水準を明記する
- 当日に大きな材料がない定番商品は、無理に入れず別テーマの候補へ入れ替える
- 逆張り候補（ベア・VIX・金・原油など）も、相場環境に合う場合は候補に含める
- 「上がりそうなもの」だけでなく、「上下どちらにも大きく振れうるもの」を選ぶ
- 各候補に **脳汁シナリオ（上振れ）** と **地獄シナリオ（下振れ）** を必ずセットで書く

### 2. 高ボラティリティ重要決算スケジュール（必須・3〜6件）

今週〜来週に予定される決算のうち、**発表前後で株価が大きく振れやすい（高ボラティリティが見込まれる）もの** を抽出する。

- 対象: AI・半導体（NVDA、AVGO 等）、高 PSR グロース（PLTR、CRWD、SNOW 等）、決算でギャップしやすい人気株、暗号資産関連株、テーマ株（量子・宇宙含む）など
- 各決算に **発表日時（JST で具体的に）**、**想定変動率（インプライド・ムーブ）の目安**、**重要度（高/中/低）** を付ける
- 想定変動率はオプション市場のインプライド・ムーブや過去の決算後変動から概算。不明なら `N/A`
- 各決算に **予想超え（▲）/ 予想未達（▼）の相場・関連銘柄への波及** と、**決算前 / 決算後の行動** を書く
- 単なる決算一覧にしない。「なぜ大きく振れるか」「どう波及するか」を必ず添える
- 経済指標（CPI・雇用統計等）はこのセクションに入れない。それは「3. 今週の重要イベント」へ

### 3. 今週の重要イベント（決算スケジュールを除く）（必須・3〜5件）

決算 **以外** の、相場全体を動かしうるイベントを **日付付き** で。

- 対象: 経済指標（米CPI / PPI / 雇用統計 / 米小売 等）、中央銀行（FOMC / 日銀会合 / FRB高官発言）、政治・地政学イベント、メジャーSQ、大型上場・指数入替 など
- **決算はここに入れない**（決算は「2. 高ボラティリティ重要決算スケジュール」へ）
- 各イベントに **発表日時（JST）**、**重要度（高/中/低）**、**上振れ（▲）/ 下振れ（▼）の相場への影響**、**イベント前 / イベント後の行動** を書く

### 4. 10バガー候補銘柄リサーチ（必須・3〜5件）

米国上場株の中から、今後5〜10年で10倍以上のリターンを狙える可能性がある企業を **調査候補として** 3〜5 件抽出する。短期の値動きや人気テーマではなく、事業成長・市場拡大・競争優位・財務健全性・バリュエーションの観点で評価する。

#### 対象ユニバース
- 米国上場株
- 時価総額は小型〜中型株を中心にする
- 極端なペニー株、流動性の低い銘柄、情報開示が乏しい銘柄は除外
- セクターは限定しないが、構造的成長市場を優先
- 毎回同じ人気銘柄に寄らないよう、候補探索時は **最低3セクター**（例: software / healthcare / industrial automation / energy transition / fintech / cybersecurity / space / robotics / quantum 等）を比較する
- 前回レポートと同じ候補は、原則 **最大2件まで**。3件以上重なる場合は、当日新たに確認した決算・ガイダンス・重要ニュース・バリュエーション変化を各銘柄に明記する

#### 探索すべき企業タイプ
- 巨大な TAM を持つ新興カテゴリのリーダー候補
- 既存産業をソフトウェア化・自動化する企業
- 高粗利・高リテンション・営業レバレッジを持つ企業
- 一時的に市場から過小評価されている構造的成長企業
- 創業者主導、または資本配分能力の高い経営陣を持つ企業

#### 一次スクリーニング条件
- 売上成長率が高い、または再加速の兆候がある
- 粗利率が高い、または改善傾向にある
- 営業利益率・FCF マージンに改善余地がある
- 過剰債務がない
- 希薄化が過度でない
- 市場規模に対して現在の売上がまだ小さい
- 競争優位性を説明できる

#### 評価フレーム
- **Market**: TAM / SAM / SOM、市場成長率、規制・技術・消費者行動の追い風、現在の売上が市場規模に対してどれほど小さいか
- **Business Quality**: 売上成長の持続性、粗利率、営業レバレッジ、顧客維持率 / NRR / 解約率、ユニットエコノミクス、収益の継続性
- **Competitive Advantage**: ネットワーク効果、スイッチングコスト、データ優位、ブランド、規模の経済、技術的差別化、規制・認証・販売網などの参入障壁
- **Financials**: 売上成長率、粗利率、営業利益率、FCF、現金残高、債務水準、株式報酬、発行済株式数の増加率
- **Management**: 創業者主導か、インサイダー保有比率、過去のガイダンス達成度、資本配分の実績、株主との利害一致、過度なプロモーションの有無
- **Valuation**: 現在の時価総額、10倍時の想定時価総額、10年後の売上・利益・FCF シナリオ、期待成長に対する現在の評価倍率、強気・標準・弱気シナリオ別の期待リターン

#### 10バガー成立条件
- 現在の時価総額から10倍になっても市場規模的に不自然でない
- 売上が5〜10倍以上になる現実的な経路がある
- 利益率改善によるマージン拡大余地がある
- 市場評価倍率の維持または再評価余地がある
- 長期で保有できるだけの事業品質がある

#### 除外条件
- 売上実態が乏しいテーマ株
- 頻繁な増資
- 過剰な株式報酬
- 粗利率が低く改善余地も乏しい
- 債務過多
- 経営陣の信頼性に問題がある
- 競争優位が説明できない
- 一過性ブームに依存している
- 開示情報が不十分

#### 参照ソース
各候補について、最低でも直近の決算資料 / 10-Q・10-K / investor relations / 信頼できるマーケットデータ / 関連ニュースを確認する。数値は取得時点とソースを `Sources` に残す。推測で評価せず、不明な項目は `N/A` と明記する。

#### 候補選定の鮮度ルール
- 候補リストは過去レポートからコピーしない。まず当日調べたスクリーニング結果・決算・ニュースから 8〜12 件のロングリストを作り、そこから 3〜5 件へ絞る
- ロングリストには、採用候補だけでなく除外候補も含め、除外理由（時価総額が大きすぎる、粗利率が低い、希薄化が重い、競争優位が弱い等）を内部的に確認する
- 採用銘柄が前回と重なる場合でも、Investment thesis / Major risks / KPIs は当日の最新情報で書き直す
- 「調査候補の継続」自体を理由にしない。継続する根拠は、事業進捗・財務・評価倍率・市場機会のいずれかで説明する

### 5. 量子コンピューティング関連ニュースおよび銘柄（必須）

過去24時間〜直近の **量子コンピューティング関連ニュース** と **関連銘柄** を集約する。

- **ニュース 2〜4 件**: 量子ハード/ソフト、誤り訂正の進展、量子優位性の実証、政府・大学・大手クラウド（IBM / Google / Microsoft / AWS 等）の動き、資金調達・提携・受注。各ニュースに **関連銘柄 / 相場への影響 / 注目点** を添える
- **関連銘柄 2〜4 件**: IONQ / RGTI / QBTS / QUBT / ARQQ / IBM などを中心に、当日の材料に応じて選ぶ（固定枠ではない）。各銘柄に **分野・時価総額（取得時点）・区分（Watchlist / Speculative 等）・強気材料・弱気材料・注目KPI** を添える
- 純粋なテーマ株は値動きが荒いため、**投機性が高い旨**を1行で添える
- 当日に目立った材料が無い場合は、ニュースは「目立った新規材料なし」と明記し、銘柄は監視対象として現状を簡潔に書く（無理に煽らない）

### 6. 宇宙関連ニュースおよび銘柄（必須）

過去24時間〜直近の **宇宙関連ニュース** と **関連銘柄** を集約する。

- **ニュース 2〜4 件**: 打ち上げの成否、新型ロケット・衛星、政府予算・契約（NASA / 国防 / 各国宇宙機関）、衛星通信・地球観測・宇宙インフラ、資金調達・提携。各ニュースに **関連銘柄 / 相場への影響 / 注目点** を添える
- **関連銘柄 2〜4 件**: RKLB / LUNR / ASTS / RDW / PL / SpaceX 関連の上場プロキシ などを中心に、当日の材料に応じて選ぶ（固定枠ではない）。各銘柄に **分野・時価総額（取得時点）・区分・強気材料・弱気材料・注目KPI** を添える
- 投機性が高い旨を1行で添える
- 当日に目立った材料が無い場合は、その旨を明記する

## Display Rules

- 重要度: **高 / 中 / 低**
- 確信度（10バガー・テーマ銘柄）: **高 / 中 / 低**
- 銘柄区分（10バガー）: High-conviction compounder / Speculative high-upside / Turnaround growth / Watchlist only / Avoid
- 上振れ: ▲ / 下振れ: ▼ / 取得不可: `N/A`

## Important Rules（絶対遵守）

- **6セクション固定**: 上記の 1〜6 の構成・並び順を必ず守る。投資判断ヒーロー・温度計・資産別アクション・総合結論などの旧セクションは作らない
- **行動への変換を最優先**: 単なるニュース要約・銘柄紹介で終わらせない。「影響 → 自分の行動 → 判断が変わる条件」をセットで書く（テーマ銘柄では強気材料・弱気材料・注目KPI で代替）
- **決算とイベントを分離**: 決算は「2.」、決算以外のイベントは「3.」に必ず分ける。混在させない
- **前日以前の内容に引きずられない**: 前回レポートの銘柄・文面・判断を土台にせず、当日取得した情報から毎回作り直す
- **候補の固定化を避ける**: 脳汁チャンス・決算・10バガー・量子/宇宙銘柄は、定番名や前回候補を自動採用しない。同じ候補を再掲する場合は当日の新材料を必ず書く
- **文面の使い回し禁止**: シナリオ・決算想定・イベント分岐・銘柄 thesis は前回表現の焼き直しにしない
- 情報源不明のニュースは採用しない
- 価格データとニュースの時刻ズレは明記
- 有料記事の本文を長く転載しない
- 出典URLを可能な限り添える
- 数値は取得時刻とセットで把握する
- 「絶対に上がる」「必ず下がる」等の **断定表現は禁止**
- ソース間で数値が食い違う場合は両方明示
- 「来週」「近々」のような **曖昧な日付表現を避ける**。日時は **JST で具体的に**
- 脳汁チャンスは **本玉での参戦は推奨しない** 旨を毎回明記
- 量子・宇宙テーマ銘柄は **投機性が高い** 旨を毎回明記
- 10バガー候補は **調査候補の抽出** として扱い、買い推奨・売買指示に見える表現を避ける
- 10バガー候補では、株価上昇ストーリーではなく **事業成長ストーリー** を重視し、強気シナリオだけでなく **弱気シナリオと除外理由** も必ず検討する
- 投資助言ではない旨の disclaimer ブロックは **書かない**（HTML 側でも描画されない）
