# Market Report Skill

過去24時間の市場情報・ニュース・掲示板センチメントを集約し、**「投資判断ダッシュボード」用** の構造化マーケットレポートを日本語で生成して `reports/YYYY-MM-DD.md` に保存する。`/publish-report` でこの markdown が HTML ダッシュボードに変換される。

## Role

個人投資家（作者本人）の **投資判断を補助する編集者兼アナリスト**。情報を網羅するのではなく、「読んだあとに今日の行動が決まる」ことをゴールとする。各情報は必ず以下の流れに変換する：

```
情報 → 相場への影響 → 自分の行動 → 判断が変わる条件
```

単なるニュース要約・価格一覧・一般的な銘柄紹介の優先度は下げる。重要なのは **「何をするか／しないか／どの条件で判断を変えるか」**。

平易な日本語を徹底する。業界用語（メガキャップ、リプライス、リスクオン/オフ 等）はそのまま使わず、易しい表現に置き換えるか初出時に注釈を付ける。英略語（CPI、PPI、FOMC など）も初出時に「米CPI（消費者物価指数）」のように日本語名を併記する。

## Pre-flight

1. JST の現在日時を取得: `TZ=Asia/Tokyo date '+%Y-%m-%d %H:%M %Z'`
2. 曜日・日本の祝日を判定。土日祝は株式市場休場のため、株式系は前営業日終値ベースとなる旨をレポート冒頭で明記
3. リポジトリルートに `reports/` が無ければ作成
4. 既に同日 (`reports/YYYY-MM-DD.md`) が存在する場合、上書きするかユーザーに確認

## Data Collection

WebSearch / WebFetch を **並列** に呼んで効率化する。

### Market Data (必須7資産)
日経平均 / S&P500 / Nasdaq / Bitcoin / ドル円 / 金 / 原油（WTI）。各項目で **現在値・前日比・騰落率・取得時刻** を取得。複数ソースで値が食い違う場合は両方明示。

### Volatility & Rates (必須4指標)
**米VIX**（米恐怖指数）／ **日経VI**（日本株の不安度）／ **米10年債利回り** ／ **日本10年債利回り**。各項目で **現在値・前日比** を取得。

### Calendar (必須)
今週〜来週の **経済指標 / 主要決算 / 政治・地政学イベント** を **日付付き** で具体的に。米CPI / PPI / 雇用統計 / FOMC / 日銀会合 / 主要企業決算（NVDA、トヨタ、東京エレクトロン 等）など。

### News (過去24時間)
経済指標 / 企業決算 / 中央銀行発言 / 地政学 / 商品市況 / 暗号資産。**価格や投資判断に影響したものに絞る**。網羅は不要。

### Bulletin Board Sentiment (WebFetch)
- 楽天日本株4.3倍ブル: https://finance.yahoo.co.jp/quote/9I31115A/forum
- 楽天日本株3.8倍ベアIII: https://finance.yahoo.co.jp/quote/9I311249/forum

各掲示板で **全体の雰囲気 / 投稿数の増減 / 感情（恐怖・楽観・諦め・FOMO）/ 頻出キーワード 3〜4 個** を要約。**個別投稿の本文を長く引用しない。要約のみ。**

### 脳汁チャンス候補
レバレッジ ETF（SOXL / TQQQ / 楽天4.3倍ブル / UVXY 等）、決算前後でボラの大きい個別株（例: NVDA）、暗号資産（BTC）など、**短期で大きく振れる対象** の現在の状況を把握。

### 10バガー候補銘柄リサーチ
米国上場株の中から、今後5〜10年で10倍以上のリターンを狙える可能性がある企業を **調査候補として** 3〜5 件抽出する。短期の値動きや人気テーマではなく、事業成長・市場拡大・競争優位・財務健全性・バリュエーションの観点で評価する。

#### 対象ユニバース
- 米国上場株
- 時価総額は小型〜中型株を中心にする
- 極端なペニー株、流動性の低い銘柄、情報開示が乏しい銘柄は除外
- セクターは限定しないが、構造的成長市場を優先

#### 探索すべき企業タイプ
- 巨大な TAM を持つ新興カテゴリのリーダー候補
- 既存産業をソフトウェア化・自動化する企業
- 高粗利・高リテンション・営業レバレッジを持つ企業
- 一時的に市場から過小評価されている構造的成長企業
- 創業者主導、または資本配分能力の高い経営陣を持つ企業

#### 一次スクリーニング条件
- 売上成長率が高い、または再加速の兆候がある
- 粗利率が高い、または改善傾向にある
- 営業利益率・FCF マージンに改善余地がある
- 過剰債務がない
- 希薄化が過度でない
- 市場規模に対して現在の売上がまだ小さい
- 競争優位性を説明できる

#### 評価フレーム
- **Market**: TAM / SAM / SOM、市場成長率、規制・技術・消費者行動の追い風、現在の売上が市場規模に対してどれほど小さいか
- **Business Quality**: 売上成長の持続性、粗利率、営業レバレッジ、顧客維持率 / NRR / 解約率、ユニットエコノミクス、収益の継続性
- **Competitive Advantage**: ネットワーク効果、スイッチングコスト、データ優位、ブランド、規模の経済、技術的差別化、規制・認証・販売網などの参入障壁
- **Financials**: 売上成長率、粗利率、営業利益率、FCF、現金残高、債務水準、株式報酬、発行済株式数の増加率
- **Management**: 創業者主導か、インサイダー保有比率、過去のガイダンス達成度、資本配分の実績、株主との利害一致、過度なプロモーションの有無
- **Valuation**: 現在の時価総額、10倍時の想定時価総額、10年後の売上・利益・FCF シナリオ、期待成長に対する現在の評価倍率、強気・標準・弱気シナリオ別の期待リターン

#### 10バガー成立条件
- 現在の時価総額から10倍になっても市場規模的に不自然でない
- 売上が5〜10倍以上になる現実的な経路がある
- 利益率改善によるマージン拡大余地がある
- 市場評価倍率の維持または再評価余地がある
- 長期で保有できるだけの事業品質がある

#### 除外条件
- 売上実態が乏しいテーマ株
- 頻繁な増資
- 過剰な株式報酬
- 粗利率が低く改善余地も乏しい
- 債務過多
- 経営陣の信頼性に問題がある
- 競争優位が説明できない
- 一過性ブームに依存している
- 開示情報が不十分

#### 参照ソース
各候補について、最低でも直近の決算資料 / 10-Q・10-K / investor relations / 信頼できるマーケットデータ / 関連ニュースを確認する。数値は取得時点とソースを `Sources` に残す。推測で評価せず、不明な項目は `N/A` と明記する。

### 補足（参考）
セクター動向や個別銘柄ニュースは、必要に応じて取得する（ニュースや資産別アクションの根拠として使う）。**画面上のセクション** にはしない。

## Display Rules

- 上昇: 🟢 / 下落: 🔴 / 横ばい: ⚪ / 取得不可: `N/A`
- 総合スタンス: **強気 / やや強気 / 中立 / やや弱気 / 弱気** の5段階
- 確信度: **高 / 中 / 低**
- 温度計の判定: **平静 / 警戒 / 高水準 / 過熱 / リスクオフ** の5段階（HTML の `.level.calm/warn/high/over/off` に対応）
- ニュース・イベントの重要度: **高 / 中 / 低**

## Important Rules（絶対遵守）

- **行動への変換を最優先**: 単なるニュース要約・価格一覧で終わらせない。「相場への影響 → 自分の行動 → 判断が変わる条件」をセットで書く
- 情報源不明のニュースは採用しない
- 価格データとニュースの時刻ズレは明記
- 掲示板コメントを **事実として扱わない**（センチメント参考のみ、逆指標として）
- 有料記事の本文を長く転載しない
- 出典URLを可能な限り添える
- 数値は取得時刻とセットで表示
- 「絶対に上がる」「必ず下がる」等の **断定表現は禁止**
- ソース間で数値が食い違う場合は両方明示
- 「来週」「近々」のような **曖昧な日付表現を避ける**。日時は **JST で具体的に**
- 脳汁チャンスは **本玉での参戦は推奨しない** 旨を毎回明記
- 10バガー候補は **調査候補の抽出** として扱い、買い推奨・売買指示に見える表現を避ける
- 10バガー候補では、株価上昇ストーリーではなく **事業成長ストーリー** を重視する
- 10バガー候補では、強気シナリオだけでなく **弱気シナリオと除外理由** も必ず検討する
- 売買禁止という強い言い方ではなく「**今日は控えたいこと**」のニュアンスで書く
- 投資助言ではない旨の disclaimer ブロックは **書かない**（HTML 側でも描画されない）
